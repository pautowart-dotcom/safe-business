Задача: экран создания первой компании при входе Super Admin

Контекст: после логина, если у пользователя нет ни одной компании
(ни в одной membership), сейчас показывается тупиковое сообщение
«У вас нет доступа ни к одной компании — обратитесь к владельцу за
приглашением». Для Super Admin это неверно — у него должна быть
возможность создать первую компанию прямо с этого экрана.

Нужно:
1. На бэкенде проверить/добавить эндпоинт типа
   POST /api/platform/companies (создание компании + автоматическое
   членство создателя как владельца этой компании). Проверить в
   platform роутере, есть ли он уже — судя по docs/task-frontend-v2.md
   эта задача там могла быть частично описана.
2. На фронтенде: если у пользователя role === 'super_admin' (или
   соответствующий признак) и нет ни одной компании — вместо
   тупикового сообщения показывать простую форму «Создать компанию»
   (название студии + всё, что обязательно в схеме companies).
3. После успешного создания — обновить контекст авторизации
   (перезапросить список компаний / выполнить select-company для
   новой компании) и перейти на Dashboard.
4. Обычным пользователям без Super Admin показывать прежнее сообщение
   про приглашение — их логику не трогать.

Прежде чем писать код, покажи текущий текст docs/task-frontend-v2.md —
там могла уже быть описана эта часть, чтобы не дублировать.
