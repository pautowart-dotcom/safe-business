РЕЗУЛЬТАТЫ ПОВТОРНОГО ТЕСТА + ЛОГИ СЕРВЕРА:

1. БАГ С РОЛЬЮ — судя по логам [invite-debug], НЕ ПОДТВЕРДИЛСЯ:
   - Тест 1: req.body = {"role":"master","invitedEmail":"artem",
     "payoutPercent":"40","branchId":""} → inserted row =
     {"role":"master", ...} → GET /memberships показывает
     "role":"master" для этой записи. Всё совпадает.
   - Тест 2: req.body = {"role":"owner","invitedEmail":"артем",
     "payoutPercent":"","branchId":""} → inserted row =
     {"role":"owner", ...} → тоже всё совпадает.
   Оба раза роль передаётся, сохраняется и отображается корректно.
   Похоже, единичный случай с "eliza" (где выбрали "Мастер", но
   показало "Владелец") был вызван чем-то внешним — например, старой
   закэшированной версией фронтенда в браузере в тот момент, а не
   багом в текущем коде. Предлагаю: убери диагностические логи
   [invite-debug], пометь баг с ролью как "не удалось воспроизвести
   на актуальном коде" и закрой этот пункт этапа. Если баг повторится
   в будущем — заведём заново с точными логами.

2. НАЙДЕНА ОТДЕЛЬНАЯ ОШИБКА В ЛОГАХ (не связанная напрямую с ролью,
   но важная): в логах PM2 есть ошибка PostgreSQL:
   "pg_strtoint32_safe" с "where: unnamed portal parameter $2 = ''"
   — то есть где-то пустая строка "" пытается быть преобразована в
   число и падает с ошибкой. Судя по req.body в логах, payoutPercent
   и branchId иногда приходят как пустая строка "" вместо null
   (например, когда роль "owner" выбрана, payoutPercent пустой; или
   когда филиал не выбран, branchId = ""). Нужно найти, в каком
   именно запросе происходит попытка привести пустую строку к
   integer, и поправить: либо на фронтенде отправлять null вместо
   пустой строки для необязательных числовых полей, либо на бэкенде
   явно обрабатывать пустую строку как null перед вставкой/использованием
   в запросе.

   скопировала ссылку в буфер обмена. НО после нажатия кнопки всё поле
   с текстом ссылки полностью исчезает с экрана — из-за этого нельзя
   скопировать её повторно. Нужно: ссылка должна оставаться видимой и
   доступной для повторного копирования постоянно — вместо скрытия
   поля показывай временное всплывающее сообщение "Скопировано!" рядом
   с кнопкой, не убирая само поле со ссылкой.

4. НОВЫЙ БАГ — поле "Филиал" в форме приглашения: выпадающий список
   показывает только "Не указан" и не даёт выбрать реальный филиал,

   реальные филиалы текущей компании и позволял выбрать один из них
   (это может быть связано с проблемой из пункта 2 — пустая строка
