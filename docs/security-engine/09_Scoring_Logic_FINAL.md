ФАЙЛ 09
ЛОГИКА РАСЧЁТА БАЛЛОВ И ГЕНЕРАЦИИ PDF
НИША: МАНИКЮР И ПЕДИКЮР
Версия 3.0 (финальная, согласованная)

ЦЕЛЬ

После прохождения платного аудита система должна:
- подсчитать общий уровень безопасности бизнеса;
- определить зону риска;
- сформировать список нарушений;
- рассчитать индекс безопасности;
- передать данные в PDF;
- сформировать персональные рекомендации.

1. СИСТЕМА БАЛЛОВ

Каждый вопрос имеет три варианта ответа (для вопросов с 4 вариантами — см. примечание ниже):

Безопасный ответ → 1 балл (Да / Есть / Оформлено / Ведётся)
Частично безопасный ответ → 0,5 балла (Частично / Не уверен / Нерегулярно / Есть сомнения)
Небезопасный ответ → 0 баллов (Нет / Отсутствует / Не ведётся)

Примечание для вопросов с 4 вариантами (например MN-502 «Подмена трудовых отношений»):
[Трудовой договор] = 1 балл
[Аренда рабочего места] = 1 балл
[Самозанятый/ГПХ] = 0,5 балла
[Не уверен] = 0 баллов

2. РАСЧЁТ ОБЩЕГО БАЛЛА (ПЛАТНЫЙ АУДИТ)

Формула:
Полученные баллы ÷ Максимально возможные баллы × 100 = Индекс безопасности (%)

Максимум баллов = количество видимых вопросов для сегмента пользователя × 1 балл.
Рассчитывается программно (см. Файл 06, раздел «Количество вопросов — программная логика»). Не фиксируется как число в документации, так как зависит от сегмента и может меняться при доработке вопросника.

Пример (пользователь с сотрудниками, фактическое число видимых вопросов — N):
Получено: X баллов из N
Расчёт: X ÷ N × 100 = индекс безопасности (%)

3. ИНДЕКС БЕЗОПАСНОСТИ — ЕДИНАЯ ШКАЛА (ИСПОЛЬЗУЕТСЯ ВЕЗДЕ: БЕСПЛАТНЫЙ И ПЛАТНЫЙ АУДИТ)

80–100% → ЗЕЛЁНАЯ ЗОНА → Высокий уровень безопасности
50–79% → ЖЁЛТАЯ ЗОНА → Средний уровень безопасности
0–49% → КРАСНАЯ ЗОНА → Высокий уровень риска

Эта шкала используется в Файле 03 (бесплатный аудит), Файле 04 (результат), Файле 08 (структура PDF) и здесь. Других шкал (10-балльной, 12-балльной отдельно от процентов) в продукте не используется.

4. РАСЧЁТ ДЛЯ БЕСПЛАТНОГО АУДИТА (12 вопросов, см. Файл 03)

Максимум: 12 баллов (если все вопросы показаны — пользователь с сотрудниками)
Если кадровые вопросы скрыты (самозанятый один) — максимум пересчитывается на фактическое количество показанных вопросов.

Формула та же: Полученные баллы ÷ Максимум показанных вопросов × 100 = Индекс безопасности (%)
Зона определяется по той же шкале из п.3.

5. ЛОГИКА СОЗДАНИЯ НАРУШЕНИЙ

Если пользователь получает 0 баллов или 0,5 балла → система создаёт нарушение.
Если пользователь получает 1 балл → нарушение не создаётся.

Исключения:
- MN-203 (Отходы класса Б): если ответ «Контакта с кровью нет» — нарушение не создаётся независимо от баллов.
- MN-205/MN-206-доп (Рециркулятор): если ответ «Рециркулятора нет» — создаётся нарушение MN-206-доп (риск 8/10), а не MN-205.
- Блок MN-501–MN-504 (Персонал): не задаётся и не создаёт нарушений, если у пользователя нет сотрудников (по правилам Файла 02).

6. ИСТОЧНИК ДАННЫХ О НАРУШЕНИЯХ

Все данные о нарушениях система берёт ИСКЛЮЧИТЕЛЬНО из Файла 10 «Матрица нарушений» (финальная версия).
Именно Файл 10 — единственный источник: уровня риска, размера штрафа, нормативной базы, стоимости устранения, срока устранения, текста рекомендации.

Файл 07 (старая короткая матрица) не используется.

7. ПОРЯДОК ФОРМИРОВАНИЯ PDF

После завершения аудита система формирует (порядок строго такой):
1. Титульный лист
2. Содержание
3. Резюме (сводная оценка бизнеса)
4. Карта уязвимостей
5. Дорожная карта устранения
6. Обязательные документы
7. Дополнительные зоны внимания
8. Персональные рекомендации
9. Какие органы могут проверять бизнес
10. Что делать дальше
11. Дисклеймер

8. ДАННЫЕ, ПЕРЕДАВАЕМЫЕ В PDF

Система передаёт:
- форму регистрации (самозанятый/ИП/ООО);
- наличие сотрудников/модель работы;
- нишу;
- ответы пользователя (бесплатный + платный аудит);
- список нарушений с полными атрибутами из Файла 10;
- общий балл (отдельно для бесплатного и платного этапа);
- индекс безопасности (%);
- статус безопасности (зона);
- суммарную стоимость устранения;
- суммарные сроки устранения;
- топ-3 нарушения.

9. ОСНОВНОЕ ПРАВИЛО РАЗДЕЛЕНИЯ ОТВЕТСТВЕННОСТИ МЕЖДУ ФАЙЛАМИ

Файл 09 отвечает ТОЛЬКО за:
- расчёт баллов;
- расчёт индекса безопасности;
- определение порядка сборки PDF;
- передачу данных между разделами.

Файл 09 НЕ содержит:
- штрафы;
- уровни рисков;
- нормативную базу;
- тексты рекомендаций.
Все эти данные — исключительно в Файле 10.
