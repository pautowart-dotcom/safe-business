АРХИТЕКТУРА БОТА
Версия 2.0 (финальная, согласованная)

1. ЦЕЛЬ ПРОДУКТА

Telegram-бот проводит экспресс-аудит бизнеса, выявляет основные риски, продаёт расширенный аудит и формирует персонализированный PDF-отчёт.

Основная задача MVP:
Получить первые платные продажи аудита и собрать обратную связь пользователей.

2. ЦЕЛЕВЫЕ НИШИ И СЕГМЕНТЫ РЫНКА

ВАЖНОЕ ИЗМЕНЕНИЕ АРХИТЕКТУРЫ (версия 2.1):
Перед выбором ниши добавляется шаг выбора сегмента рынка. Структура: Сегмент → Ниша → Вопросы.
Это не усложняет MVP — добавляет один экран в сегментацию и одно поле в базу данных.
Позволяет правильно масштабировать систему на любые новые категории бизнеса в будущем.

ШАГ ВЫБОРА СЕГМЕНТА (новый экран между «Форма работы» и «Выбор ниши»):

Текст: «Выберите сферу деятельности»

Варианты на MVP:
- Красота и здоровье (маникюр, брови/ресницы, волосы, массаж)
- Фитнес и активность (фитнес, танцы, йога)
- Розничная торговля (цветы, одежда, товары)
- Общепит (кафе, кофейня, доставка)
- Другое (лист ожидания)

После выбора сегмента — показывается список ниш внутри этого сегмента.

ТЕКУЩИЙ СТАТУС НИШ ПО ГЛУБИНЕ РЕАЛИЗАЦИИ:

Сегмент «Красота и здоровье»:
- Маникюр и педикюр — ПОЛНОСТЬЮ ГОТОВА: бесплатный аудит, платный аудит (Файл 06), матрица нарушений (Файл 10), структура PDF. Единственная ниша с готовым платным продуктом на старте.
- Ресницы и брови — бесплатный тест работает, платный аудит — ЗАГЛУШКА.
- Волосы — бесплатный тест работает, платный аудит — ЗАГЛУШКА.
- Массаж — бесплатный тест работает, платный аудит — ЗАГЛУШКА.

Все остальные сегменты и ниши — ЗАГЛУШКА на все продукты включая бесплатный тест (см. сценарий ниже).

СЦЕНАРИЙ ДЛЯ НИШ БЕЗ ПЛАТНОГО КОНТЕНТА:
При нажатии «Расширенный аудит» показывается:
«Расширенный аудит для ниши [название] в разработке. Мы вас уведомим о запуске.»
Пользователь сохраняется в лист ожидания.

СЦЕНАРИЙ ДЛЯ СЕГМЕНТОВ БЕЗ БЕСПЛАТНОГО ТЕСТА (все кроме «Красота и здоровье» и «Фитнес»):
При выборе показывается:
«Сейчас эта сфера деятельности ещё не поддерживается. Мы можем уведомить вас после запуска.»
Данные сохраняются в лист ожидания.

МУЛЬТИВЫБОР НИШИ — ЗАПЛАНИРОВАННАЯ БУДУЩАЯ ФУНКЦИЯ (не входит в текущий спринт):

Контекст: часть пользователей ведут несколько направлений одновременно (например, салон красоты = маникюр + брови + волосы). Мультивыбор позволит проходить один аудит без дублирования вопросов и нарушений.

Логика (для реализации в следующей версии):
1. Пользователь выбирает основную нишу (обязательно) и дополнительные ниши (опционально, до 3).
2. Каждый вопрос в матрице помечается тегами ниш, к которым относится (поле «niches» в базе, массив значений).
3. Бот задаёт вопросы основной ниши полностью. Из дополнительных ниш — только те вопросы, которых ещё не было (дедупликация по тегу).
4. В PDF нарушения из разных ниш объединяются в одну таблицу без дублей, сортируются по риску.

Почему не сейчас: требует разметки тегов на всех 29 нарушениях матрицы и логики дедупликации — отдельная работа, которую целесообразно делать после подтверждения спроса и накопления данных о том, насколько часто пользователи ведут несколько ниш.

4. ЭКРАН ПРИВЕТСТВИЯ

Текст:
Здравствуйте.
Меня зовут Артём, я основатель сервиса «Безопасный Бизнес».
Мы помогаем владельцам студий и кабинетов находить риски до того, как ими заинтересуются проверяющие органы.

За несколько минут вы получите:
• оценку уровня безопасности бизнеса;
• список основных рисков;
• рекомендации по устранению нарушений;
• предварительную оценку возможных последствий.

Мы не собираем названия компаний, адреса и паспортные данные.
Все ответы используются только для формирования отчёта.

Кнопки:
[Начать аудит]
[О сервисе]
[Политика конфиденциальности]

Текст:

7. ЛОГИКА ПОКАЗА ВОПРОСОВ

Полная логика — в Файле 02 «Сегментация пользователей и правила показа вопросов».

Краткий принцип:
Кадровые вопросы (трудовые договоры, медкнижки, инструктаж, СОУТ, подмена трудовых отношений) показываются только если у пользователя есть сотрудники или смешанная модель работы.

8. БЕСПЛАТНЫЙ АУДИТ

12 вопросов, единые для всех ниш и сегментов (вопросы из кадрового блока скрываются по правилам Файла 02).
Состав вопросов, логика ответов и расчёты — в Файле 03.

9. РЕЗУЛЬТАТ БЕСПЛАТНОГО АУДИТА

• три главных риска;

Логика — в Файле 04.

10. ПРОДУКТЫ ПОСЛЕ БЕСПЛАТНОГО АУДИТА

Три кнопки: Расширенный аудит / Пакет документов / Подписка «Спокойствие».
Логика — в Файле 05.

11. РАСШИРЕННЫЙ АУДИТ

После оплаты запускается платный аудит по выбранной нише — но ТОЛЬКО для ниши «Маникюр и педикюр» на этапе MVP (см. п.2 выше).
Для остальных 4 ниш кнопка «Расширенный аудит» ведёт на заглушку с листом ожидания, оплата не запрашивается.

Количество вопросов рассчитывается программно в зависимости от сегмента пользователя (наличие сотрудников и т.д.) — см. Файл 06, раздел «Количество вопросов — программная логика». Ориентировочно от 16–19 (без сотрудников) до 23–27 (с сотрудниками) для ниши «маникюр и педикюр» — точное число не фиксируется и пересчитывается автоматически при изменении вопросника.
Подробное описание — в Файле 06.

14. MVP-ОГРАНИЧЕНИЯ

На этапе MVP:
• нет личного кабинета;
• нет CRM;
• нет повторных проверок;
• нет подписки (только лист ожидания);
• нет автоматического обновления законодательства;
• нет раздела «Проверенные партнёры» (до появления реальных партнёров).

Только аудит и PDF.
