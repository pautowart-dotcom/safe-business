ФАЙЛ 02
СЕГМЕНТАЦИЯ ПОЛЬЗОВАТЕЛЕЙ И ПРАВИЛА ПОКАЗА ВОПРОСОВ
Версия 2.0 (финальная, согласованная)

ЦЕЛЬ

Перед началом аудита система должна определить тип бизнеса пользователя.
Это необходимо для того, чтобы не задавать нерелевантные вопросы и не показывать нарушения, которые физически не могут относиться к конкретному бизнесу.

1. ПАРАМЕТРЫ СЕГМЕНТАЦИИ

Шаг 1. Как оформлена деятельность?
• Самозанятый
• ИП
• ООО

Шаг 2. Как вы работаете?
• Работаю один
• Есть сотрудники
• Сдаю рабочие места
• Смешанная модель

Шаг 3. Выберите сферу деятельности [НОВЫЙ ШАГ]
• Красота и здоровье
• Фитнес и активность
• Розничная торговля
• Общепит
• Другое

Логика для шага 3:
— Если выбрано «Другое» → показать сообщение «Сейчас эта сфера не поддерживается, уведомим о запуске» → сохранить в лист ожидания → завершить.
— Если выбрано «Розничная торговля» или «Общепит» → на текущем этапе тоже заглушка → лист ожидания.
— Если выбрано «Красота и здоровье» или «Фитнес и активность» → переход к шагу 4.

Шаг 4. Выберите нишу (список зависит от сегмента)

Для сегмента «Красота и здоровье»:
• Маникюр и педикюр
• Ресницы и брови
• Волосы (парикмахерские услуги)
• Массаж (без медицинской лицензии)

Для сегмента «Фитнес и активность»:
• Фитнес-студия / тренажёрный зал
• Танцы
• Йога / растяжка

Все варианты кроме «Маникюр и педикюр» на текущем этапе ведут к заглушке на платный аудит (бесплатный тест работает для всех).

МУЛЬТИВЫБОР НИШИ — ЗАПЛАНИРОВАННАЯ БУДУЩАЯ ФУНКЦИЯ

На текущем этапе пользователь выбирает одну нишу.
В будущей версии будет возможность выбрать несколько ниш (например, салон красоты = маникюр + брови + волосы).
Логика дедупликации вопросов и нарушений описана в Файле 01, раздел 2.
Реализуется после накопления данных о частоте мультиниш среди пользователей.

2. ЛОГИКА ПОКАЗА ВОПРОСОВ

Каждый вопрос имеет статус:
• Показывать всегда
• Показывать только определённым сегментам
• Скрывать

3. БЛОК «КАДРЫ»

Вопросы:
• Трудовые договоры (M001 в бесплатном / MN-504 в платном — см. Файл 06, поле «Инструктаж по охране труда» относится отдельно к MN-504; трудовые договоры как таковые в платном тесте не переспрашиваются отдельным вопросом — они покрываются вопросом MN-502 «Подмена трудовых отношений», где вариант ответа [Трудовой договор] = 1 балл)
• Медицинские книжки сотрудников (M002 / MN-503)
• Инструктаж по охране труда (M003 / MN-504)
• Подмена трудовых отношений (MN-502)

Показывать:
ИП + сотрудники
ООО + сотрудники
ИП + смешанная модель
ООО + смешанная модель

Не показывать:
Самозанятый + работаю один
Самозанятый + сдаю рабочие места
ИП + работаю один
ООО + работаю один

4. БЛОК «СОУТ»

Вопрос: Проводилась ли специальная оценка условий труда? Код в матрице и в платном тесте — MN-501 (единый код, без расхождений).

Показывать: Если есть сотрудники.
Скрывать: Самозанятый. Работаю один.

5. БЛОК «ПОДМЕНА ТРУДОВЫХ ОТНОШЕНИЙ»

Вопрос: Каким образом оформлены специалисты, работающие в помещении? (MN-502)

Показывать:
Если есть сотрудники.
Если смешанная модель.
Если есть мастера на договорах (сдаю рабочие места).

Скрывать:
Самозанятый, работающий один.

6. БЛОК «МЕДИЦИНСКИЕ КНИЖКИ»

Показывать: Если есть сотрудники.
Скрывать: Если пользователь работает один без сотрудников.

Важное уточнение (добавлено в финальной версии):
Если пользователь — самозанятый и сдаёт рабочие места другим мастерам, вопрос о медкнижках арендаторов НЕ показывается этому пользователю — это ответственность самого арендатора, а не арендодателя. Этот нюанс передаётся в раздел «Дополнительные зоны внимания» как рекомендация проверить у арендаторов.

7. БЛОК «ОБЩИЕ ВОПРОСЫ»

Показываются всегда, независимо от сегмента:
• Публичная оферта
• Персональные данные
• Фото и видео клиентов
• Пожарная безопасность
• Аптечка
• Рециркулятор
• Уведомление о начале деятельности
• Приём безналичной оплаты / эквайринг (MN-103) [добавлено]
• Онлайн-касса
• Защита клиентской базы
• Договор аренды (нишевый блок MN-101)
• Все санитарные вопросы (MN-201 — MN-209)
• Все вопросы по оборудованию (MN-301 — MN-305)
• Все вопросы по персональным данным (MN-401 — MN-404)
• Все вопросы по эксплуатации помещения, включая пожарную сигнализацию (MN-601 — MN-606) [обновлено]
• Дополнительные зоны (MN-701 — MN-703)

Удалено из финальной версии: вопрос про лицензионное ПО (M011) — заменён на вопрос про договор аренды в бесплатном тесте (см. Файл 03).

8. ЛОГИКА СОЗДАНИЯ НАРУШЕНИЙ

Если вопрос не показывался пользователю, связанное нарушение не может быть создано.

Пример:
Самозанятый. Вопрос про трудовые договоры не показывался. Нарушение M001 не создаётся, даже если в матрице оно существует.

9. ЛОГИКА ДЛЯ PDF

В итоговый PDF включаются только нарушения, относящиеся к конкретному пользователю.
Неиспользуемые блоки скрываются полностью.

Пример:
Самозанятый мастер маникюра, работает один.
В PDF отсутствуют:
• трудовые договоры;
• СОУТ;
• кадровые журналы;
• приказы о приёме;
• охрана труда сотрудников;
• подмена трудовых отношений.

10. ЛОГИКА ДЛЯ СПИСКА ОБЯЗАТЕЛЬНЫХ ДОКУМЕНТОВ

Список документов также зависит от сегмента (подробности — Файл 13).

Пример. Самозанятый мастер маникюра, работает один.
Показывать: согласие на ПД, согласие на фото/видео, политика конфиденциальности, сертификаты на оборудование, журнал стерилизации, журнал уборки, журнал рециркулятора.
Не показывать: трудовые договоры, приказы, СОУТ, кадровые инструкции.

Пример. ООО с сотрудниками.
Показывать полный перечень документов.

11. ПРАВИЛО MVP

Любая новая ниша, добавляемая в будущем, обязана использовать эту же систему сегментации.
Запрещено создавать отдельную логику для каждой ниши.
Сначала определяется тип бизнеса. Потом показываются соответствующие вопросы. Только после этого применяется нишевая логика.

12. ИТОГОВОЕ КОЛИЧЕСТВО ВОПРОСОВ ПЛАТНОГО АУДИТА (МАНИКЮР) ПО СЕГМЕНТАМ

Количество вопросов не фиксируется как число в документации — оно рассчитывается программно как количество видимых вопросов для конкретного сегмента (подробная логика — Файл 06, раздел «Количество вопросов — программная логика»).

Принцип:
Без сотрудников (самозанятый/ИП, работает один) — блок «Персонал» (4 вопроса MN-501–504) скрыт, остальные блоки показаны полностью.
С сотрудниками (ИП/ООО, есть сотрудники или смешанная модель) — показывается полный список вопросов всех блоков.

Точный список вопросов по сегментам и блокам — Файл 06.
