КОЛИЧЕСТВО ВОПРОСОВ — ПРОГРАММНАЯ ЛОГИКА (НЕ ФИКСИРОВАННОЕ ЧИСЛО)

Важно для разработчика: итоговое количество вопросов в платном аудите НЕ задаётся как константа в ТЗ. Оно рассчитывается программно:

количество_вопросов = count(вопросы из блоков 1–7, где show_if(сегмент_пользователя) == true)

Правило показа блока 4 «Персонал» (4 вопроса: MN-501–504) — по правилам Файла 02: показывается только если у пользователя есть сотрудники или смешанная модель работы. Все остальные блоки (1, 2, 3, 5, 6, 7) показываются всем сегментам всегда.

Причина такого подхода: при добавлении, удалении или изменении условий показа любого вопроса в будущем (новая ниша, новый сегмент, правка вопросника) количество вопросов меняется автоматически и нигде не приходится синхронизировать число вручную. Любая зафиксированная в тексте ТЗ цифра рискует разойтись с фактической реализацией при следующей правке — поэтому она здесь не приводится.

Если для дизайна экрана прогресса («Вопрос 5 из …») нужно знать число до начала теста — оно вычисляется тем же способом сразу после прохождения сегментации (Файл 02), до показа первого вопроса.
