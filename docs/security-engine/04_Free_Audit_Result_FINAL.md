ЦЕЛЬ БЛОКА

После прохождения бесплатного аудита пользователь должен получить:
- Понятный результат.
- Ощущение ценности.
- Осознание рисков.
- Желание получить полную картину.
- Переход к покупке расширенного аудита.

1. РАСЧЁТ ЗОНЫ БЕЗОПАСНОСТИ

Используется единая шкала из Файла 09 (по процентам, не по баллам):
80–100% → ЗЕЛЁНАЯ ЗОНА
50–79% → ЖЁЛТАЯ ЗОНА
0–49% → КРАСНАЯ ЗОНА

2. ЛОГИКА ВЫБОРА ТРЁХ ГЛАВНЫХ РИСКОВ

Система анализирует только вопросы, по которым пользователь получил 0 баллов или 0,5 балла.
Если нарушений больше трёх — сортировка по приоритету (таблица ниже).
Если нарушений меньше трёх — показываются все имеющиеся.

4. ДАННЫЕ ДЛЯ КАЖДОГО НАРУШЕНИЯ

M001 — Трудовые договоры
Описание: Возможны претензии трудовой инспекции и налоговых органов.

M002 — Медицинские книжки
Описание: Возможны штрафы и предписания Роспотребнадзора.

M003 — Охрана труда
Описание: При отсутствии документов возможны штрафы при проверке ГИТ.

M004 — Публичная оферта
Описание: Повышается риск конфликтов и возвратов денежных средств.

M005 — Персональные данные
Описание: Возможны штрафы за нарушения требований законодательства о персональных данных.

M006 — Фото и видео клиентов
Описание: Публикация без согласия может привести к претензиям клиента.

M007 — Пожарная безопасность
Описание: Возможны штрафы и предписания по устранению нарушений.

M008 — Аптечка и санитарная безопасность
Описание: Повышается риск претензий со стороны контролирующих органов.

M009 — Уведомление о начале деятельности
Описание: Возможны санкции за отсутствие обязательного уведомления.

M010 — Кассовая дисциплина
Описание: Возможны штрафы за нарушения порядка расчётов.

M011 — Договор аренды [ЗАМЕНЕНО — было «Лицензионное ПО»]
Описание: Несоответствие вида деятельности в договоре аренды может привести к штрафу и риску ограничения деятельности.

M012 — Защита клиентской базы
Описание: Повышается риск утечки персональных данных клиентов.
